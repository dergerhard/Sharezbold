Sharezbold-Applikation

In diesem Abschnitt finden Sie die Bedienung der Sharezbold Software. Mit dieser software kann eine Migration von SharePoint 2010 auf SharePoint 2013 oder SharePoint 2013 auf SharePoint 2013 durchgeführt werden.

Installer

Beachten Sie: Für die Installation sind Admin-Benutzerrechte erforderlich!


Zunächst muss die Sprache ausgewählt werden.

Im nächsten Schritt starten man den Prozess mit klick auf Weiter.

Nun wählt man das Zielverzeichnis aus und starten den Installer mit klick auf Installieren.

Zuletzt, wenn das Installieren erfolgreich war, kann das Programm gestartet werden.

In der Eingabemaske können die Einstellungen für einen erfolgreichen Migrationsprozess vorgenommen werden. Nachfolgend eine Erklärung der einzelnen Werte:
Einstellungsoption Beschreibung

Username Username für den Quell-SharePoint
Domain Domäne des Quell-SharePoints
Password Passwort des Users
Destination Configuration
Host Adresse des Ziel-SharePoints
Central Admin-URL Adresse der Administration des Ziel-SharePoints
Username Username für den Zugriff auf den Ziel-SharePoint
Domain Domäne des Ziel-SharePoints
Password Passwort des Users
Proxy
Activate Proxy-Einstellungen aktivieren
Proxy URL URL des Proxy
Username Benutzername

Miscellaneous
A site collection will be migrated Checkbox, wenn komplette SiteCollection migriert werden soll
File-Migration Service URI URI des FileMigrationService (wird für die File-Migration benötigt.)
Bandwith Bandbreite für Down- und Upload von Dokumenten
Number of Threads Anzahl an Threads für den Down- und Upload

Profile können auch gespeichert und geladen werden. Hierzu kann man das Menü File verwenden:
  • Mit Save Migration Profile ... können die aktuellen Einstellungen gespeichert werden.
  • Mit Load Migration Profile ... können Einstellungen wieder geladen werden. Die Einstellungen werden in eine XML-Datei (im Klartext) gespeichert.
Klickt man auf den Button next, so wird man intuitiv Step by Step durch das Programm gef√ºhrt. Nachfolgend sind die einzelnen Schritte kurz beschrieben:
  1. Zu Beginn kann man sich bei der Elements Migration entscheiden, welche Typ-Elemente (ContentTypes)migriert werden sollen. Zur Auswahl stehen:
    • User: Benutzer werden migriert.
    • Group: Benutzergruppen werden migriert.
    • PermissionLevels: Die Rechte werden migriert.
    • SiteColumns: Die Site-Columns werden migriert.
    • Workflow: Die Workflows werden migriert.
    Hinweis: Bestehende Typ-Elemente werden nicht gelöscht. Mit Klick auf Migrate Content-Type werden alle Typ-Elemente ausgewählt.
    Mit Klick auf Content geht's weiter zur Content-Selection.
  2. Nun kann mittels eines Tree-Pickers ausgewählt werden, welche Seiten und welcher Inhalt auf den Seiten migriert werden soll.
    Hinweis: Die erste Site trägt den selben Namen wie die SiteCollection, da die erste Site den selben Namen hat wie die SiteCollection selbst.
    Mit Klick auf Configure File Migration kommt man zur Informationsseite der File-Migration.
  3. Es folgt eine Übersicht der Informationen zur File-Migration. Aufgeführt werden punkte wie:
    1. Webs, von denen die Files migriert werden.
    2. Die ausgewählte Bandbreite.
    3. Die Anzahl an Threads.
    4. Die URI des FileMigrationService
    5. Die limitierte FileSize per Upload.
    Hinweis: Auf dieser Seite sind keine %Auml;nderungen mehr möglich. Sie dient lediglich als Info-Seite.
    Ein Klick auf Configure Migration Elements führt einem zur Seite, wo die zu migrierenden Elemente ausgewählt werden können.
  4. Auf dieser Seite gibt es zwei Boxen. Bei der linken Box können die Elemente, welche Migriert werden soll, ausgewählt werden. Darunter fallen zum Beispielen Themen wie Kalender oder Notizen. Auf der rechte Seite wird ausgewählt in welches Web migriert werden soll.
    Hinweis: Da hier viele Daten geladen werden müssen, kann der Ladevorgang einiges an Zeit in Anspruch nehmen.
    Mit Klick auf Start Migration wird die Migration gestartet.
  5. Während der Migration werden laufend Informationen ausgegeben, welche Teile erfolgreich migriert wurden. Schlägt jedoch zum Beispiel bei der Migration eines Content-Types etwas fehl, so wird das ebenfalls aufgeführt und kann im selbst korrigiert werden.
    Hinweis: Die Dauer der Migration hängt von der Anzahl der ausgewählten Elmente aus.
    Das Log wird ebenfalls ins User-Home geschrieben, damit der Benutzer nähere Informationen über die Migrationserfolg oder -misserfolg erhählt.
  6. Wenn der Migrations-Prozess abgeschlossen ist, erscheint eine Info-Textbox. Das Programm kann mit Finish Exit verlassen werden.
